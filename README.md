# 生命靈數占卜

## 大運（付費版）

以九年為一步，自出生推至 120 歲共 14 步。
大運數 = 主命數 + 步數，歸一至 1–9，與流年形成大小雙循環。

事件推導分三個年齡層，避免年齡錯置：
- 未滿 18 歲：談性格養成與家庭環境，不談婚姻財務
- 18–71 歲：完整人生事件（貴人、感情、車關、健康、財運、事業）
- 72 歲以上：談健康、傳承與心境

## 數字檢測（付費版）

頁面結構：置頂為不可更動的生日主體 → 使用說明 → 自由填寫欄位
→ 「開始測算」按鈕 → 隱私警語 → 底部整合結論。

欄位全部自由填寫：左欄自行命名、右欄輸入數字，
初始 5 列，可自行新增至 10 列，也可刪除多餘的列。
英文字母會自動略過（例如身分證 M121864569 只取 121864569）。

不預設項目名稱，是因為每個人手上的數字組合差異很大——
有人三支門號、有人多個帳戶，固定選項反而綁手綁腳。

## 單一檔案・執行時解鎖

只有 **index.html** 一個入口檔。免費與付費不再是兩份檔案，
而是同一份程式的兩種執行狀態（`unlocked` 變數）。

流程：免費 8 頁，每章底部有一段隱藏內容（顯示標題與勾引文字，
內容以模糊條遮住）→ 最後一頁按下解鎖 → 確認後就地重建報告，
回到第一頁，頂端出現金色橫幅，各章的隱藏段落隨之展開並亮起。

### ⚠️ 目前為開發／自審模式

`src/app.js` 中的 `DEV_UNLOCK_OPEN = true`：
按下解鎖會照常顯示付費確認視窗，但確認後**直接解鎖**，
方便反覆檢視付費內容，不必開兩個檔案。

**正式上線前必須改為 `false`**，並在該處串接
StoreKit（iOS）或 Play Billing（Android）的購買驗證，
驗證通過後才呼叫 `doUnlock()`。

### 各章的隱藏段落

| 章節 | 隱藏什麼 |
|---|---|
| 命格總覽 | 你現在走到人生的哪一步（大運現況） |
| 你是什麼樣的人 | 同樣這個性格，在感情裡是什麼樣子 |
| 先天數字與磁場 | 這些磁場正在怎麼影響你的錢 |
| 兩套命盤的交會 | 知道要補，那該補在哪裡 |
| 天生缺乏什麼 | 缺這個數字，會在哪裡跌倒 |
| 人際與事業 | 什麼時候會遇到那個人 |
| 流年與開運 | 未來五年，哪一年會翻身 |

隱藏的內容都是從付費章節取出的真實片段，不是佔位文字——
解鎖後顯示的就是同一份資料的完整版。

## 兩個版本

- demo.html    免費版（8 頁，關鍵處收束導向銷售頁）
- unlock.html  付費版（14 頁，含桃花、流年、命運走向、大運 0–120 歲、數字檢測與完整總表）

兩者共用 src/app.js 與 src/app.css，只差入口檔的 window.UNLOCKED 旗標。
日後修改只需改 app.js，兩版自動同步。

⚠️ 入口檔請使用 window.UNLOCKED = true/false，
   不要改成 const 或 let——會與 app.js 的宣告衝突而導致整頁空白。

## 字型（全部為 SIL OFL 1.1 開源授權，可自由商用）

| 用途 | 字型 | 供應方式 |
|---|---|---|
| 主標題 | 台北黑體 Taipei Sans TC | 需自行安裝（可選） |
| 副標題・內文 | 思源黑體 Noto Sans TC | Google Fonts 自動載入 |
| 水晶球數字 | Dela Gothic One | Google Fonts 自動載入 |

思源黑體與 Reggae One 已由 Google Fonts CDN 自動供應，不需做任何事。

台北黑體不在 Google Fonts 上，建議自行到「翰字鑄造 JT Foundry」
免費下載安裝，標題會更接近印刷風格黑體。未安裝也能正常顯示，
只是標題會退回思源黑體（仍以字重、字級、顏色區分層級）。

詳細授權說明與替換方式見 assets/fonts/README.md。

## 背景音樂

放置 assets/audio/bgm.mp3 即可自動循環播放。
於「輕觸水晶球」時啟動（瀏覽器禁止未經互動的自動播放），
有旁白的影片播放時會自動降低音量，左上角有靜音鈕。
未放置檔案時靜音鈕會自動隱藏，不影響其他功能。

## 影片已內含

assets/video/ 內已放入四支動畫，assets/img/ 內有對應首幀圖。
規格：H.264、720×1280（9:16）、24fps、含音軌。

音軌已用 EBU R128 標準正規化至 -18 LUFS（真實峰值 -1.5 dBTP）。
正規化前四支相差 15.6 dB（v1 為 -36.7、v3 為 -21.1 LUFS），
v1 幾乎聽不見，且無論怎麼調 volume 參數都補不回來——
volume 上限為 1.0，而 v1 需要 +18 dB。
正規化後彼此差距僅 0.4 dB，單一 VID_VOLUME 即可控制全部。
原始音軌備份於容器外，如需還原請重新提供原始檔。
v1/v2/v4 各 10 秒；v3 已剪短為 2.2 秒（取原始檔第 3.5～5.7 秒，
首尾各 0.35 秒淡入淡出），原始檔保留為 v3-reveal-original.mp4。

剪法說明：原始 v3 前段鋪陳過長，爆光要到第 5 秒才出現。
若依直覺「砍掉最後三秒」，爆光與數字都會被剪掉，揭曉就沒有了。
因此改為保留爆光、剪去前段鋪陳——長度同樣縮短，但高潮完整保留。

熱區座標與揭曉時機皆已依實際影片校準：
- 水晶球熱區　x 50% / y 65% / 直徑 22%（依亮度剖面實測，光環貼合球體外緣）
- 數字浮現　　1.35 秒（v3 爆光峰值為 1.5 秒）
- 轉場報告　　由 v3 的 ended 事件觸發（2.2 秒），計時器僅作保險
- 轉入報告時為硬切（不做交叉淡入），因 v3 結尾已自帶淡出
- 數字位置　　x 50% / y 48%（對準爆光中心，非水晶球位置）

若更換影片，按鍵盤 C 可開啟校準模式重新對位，
時間常數在 src/app.js 上方（REVEAL_NUM_MS / REVEAL_END_MS）。

## 生日輸入

八碼填妥且日期合法後，約 1 秒自動進入占卜，不需按任何按鈕。
輸入框底線會走一圈作為倒數提示，期間任何修改都會取消重數。
仍可直接點水晶球或按 Enter 立即送出。

## 試玩方式（一定要用本機伺服器）

直接雙擊 index.html 會失敗——瀏覽器的 file:// 協定不允許網頁載入本機影片。
必須這樣開：

1. 在本資料夾的檔案總管路徑列打 cmd 按 Enter
2. 輸入：py -m http.server 8000
3. 瀏覽器開啟：http://localhost:8000/index.html

黑視窗要保持開著。要關閉伺服器按 Ctrl+C。
（找不到 py 就改用 python，或雙擊 start-server.bat）

## 資料夾可以隨便搬

所有路徑都是相對的，整個資料夾放哪、叫什麼名字都能跑，
只要 index.html、src/、assets/ 三者維持在同一層即可。

## 調整水晶球熱區位置

1. 開啟頁面後按鍵盤 C，熱區會顯示成金色虛線圓
2. 看它有沒有蓋住影片裡的水晶球
3. 沒對準就編輯 demo.html 最上方的 HOTSPOTS：
       x = 圓心橫向位置（佔畫面寬度 %）
       y = 圓心縱向位置（佔畫面高度 %）
       size = 圓的直徑（佔畫面寬度 %）
4. 重新整理，再按 C 檢查

## 結果頁的「日／月」返回熱區

預設關閉，因為 v4 影片右上角未必真的有太陽。
若你的影片有，把 demo.html 裡的 SUN_HOTSPOT_ENABLED 改成 true 即可。
關閉時，結果頁下方會有「重新測算」文字連結。

## 其他可調參數（都在 demo.html 最上方）

CALC_MS   計算動畫長度，預設 9000 毫秒
PROGRESS  計算過程的進度文字

## 完整流程時序

    開場 v1        點水晶球（記住音樂意圖、解除自動播放限制）
      ↓
    輸入生日        八碼填妥後 1 秒自動送出
      ↓
    計算 v2        9 秒
      ↓
    揭曉 v3        2.2 秒（1.35 秒數字浮現，1.5 秒爆光）
      ↓            由 v3 的 ended 事件觸發
    命格總覽        報告第 1 頁，v4 於背景播放
      ↓            由 v4 的 ended 事件觸發
    BGM 淡入        4 秒緩起，之後無限循環

整條鏈路由影片的 ended 事件串接，非固定計時器，
避免影片更換或載入延遲造成斷層。計時器僅作保險。

## 兩套流派並存

本專案同時使用兩套數字系統，互相印證而非取代：

1. 生命靈數（西洋數字學）— 全部數字相加歸一，看人格原型
   檔案：src/engine.js、src/content.js
2. 數字易經・八星磁場（源自易經八卦）— 兩兩相鄰拆解，看能量分布
   檔案：src/magnetic.js、src/magnetic-content.js

八星磁場的價值在於可套用到後天數字（手機、車牌、住址），
因此付費後的「數字檢測工具」是回購與口碑的主要來源。

註：八星磁場的拆解方式各流派略有差異，本專案採
「剔除 0 與 5 後，相鄰兩位數滑動配對」，理由見 magnetic.js 註解。
